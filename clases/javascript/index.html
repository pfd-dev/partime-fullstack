JavaScript

JavaScript es un lenguaje de programación que permite agregar interactividad y dinamismo a las páginas web. Se ejecuta en el navegador y es utilizado para crear efectos visuales, validar formularios, manipular elementos HTML y mucho más.

¿Cómo ejecutar JavaScript?

JavaScript se puede ejecutar en una página web de tres maneras:

  • En línea: Dentro de etiquetas HTML usando el atributo onclick u otros eventos.
  • Interno: Dentro de la etiqueta <script> en el mismo archivo HTML.
  • Externo: En un archivo .js separado y enlazado con <script src="archivo.js"></script>.

Se recomienda escribir JavaScript en un archivo externo para mantener el código organizado y reutilizable.

Sintaxis básica

JavaScript tiene una sintaxis simple y flexible. Aquí algunos aspectos clave:

  • Sentencias: Cada instrucción suele terminar con un punto y coma (;), aunque no es obligatorio.
  • Comentarios: Se pueden escribir comentarios con // comentario en una línea o /* comentario en varias líneas */.
  • Case sensitive: JavaScript distingue entre mayúsculas y minúsculas. miVariable y mivariable son diferentes.
  • Indentación: se refiere a la técnica de agregar un espacio inicial (indentado) al principio de las líneas de código, que ayuda a delimitar visualmente los bloques y estructuras de control.

Respetar estas reglas es fundamental para escribir código limpio y funcional.

Tipos de datos

En JavaScript, los datos se clasifican en diferentes tipos según su naturaleza. Los principales tipos de datos son:

  • String (Cadenas de texto): Representan texto y se escriben entre comillas. Ejemplo: "Hola, mundo".
  • Number (Números): Representan valores numéricos, ya sean enteros o decimales. Ejemplo: 42, 3.14.
  • Boolean (Booleanos): Representan valores de verdadero (true) o falso (false).
  • Undefined: Una variable que ha sido declarada pero no tiene valor asignado.
  • Null: Representa la ausencia intencionada de un valor. Es un valor primitivo que indica la ausencia de valor.
  • Object (Objetos): Estructuras de datos que pueden contener múltiples valores en forma de pares clave-valor. Los objetos pueden almacenar cualquier tipo de dato.
  • Array (Arreglos): Un tipo especial de objeto que almacena una colección de valores. Los elementos de un arreglo se acceden mediante índices numéricos.
  • Symbol: Un tipo de dato primitivo introducido en ECMAScript 6. Los símbolos son valores únicos e inmutables que pueden ser utilizados como claves de propiedades de objetos, lo que ayuda a evitar colisiones de nombres entre propiedades de objetos.
  • BigInt: Un tipo de dato que permite representar números enteros que son demasiado grandes para ser representados como valores de tipo Number. Ejemplo: 9007199254740991n.
  • Function: Aunque no es un tipo de dato "primitivo", las funciones en JavaScript son objetos de primera clase y se pueden tratar como valores. Ejemplo: function miFuncion() {}.

JavaScript es un lenguaje de tipado dinámico, lo que significa que no es necesario declarar el tipo de una variable, ya que puede cambiar en tiempo de ejecución.

Variables

Una variable es un espacio en memoria donde se almacena un valor. En JavaScript, existen tres formas principales de declarar variables:

  • var: Forma tradicional de declarar variables, pero tiene un alcance menos controlado.
  • let: Se usa para declarar variables cuyo valor puede cambiar. Tiene un alcance limitado al bloque donde se declara.
  • const: Se usa para declarar constantes, es decir, valores que no cambiarán durante la ejecución del programa.

Es recomendable usar let y const en lugar de var para evitar errores y hacer el código más seguro.

Buenas prácticas para nombrar variables

  • Camel Case: Usar minúsculas para la primera palabra y mayúsculas para cada palabra subsecuente. Ejemplo: miVariable, nombreCompleto, cantidadDeArticulos.
  • Snake Case: Separar las palabras con guiones bajos. Ejemplo: nombre_completo, cantidad_de_articulos. Se usa generalmente para constantes o cuando se importa código de otros lenguajes.
  • Uso de Mayúsculas: Las constantes se escriben completamente en mayúsculas y se separan con guiones bajos. Ejemplo: PI, MAX_LONGITUD, NUMERO_DE_USUARIOS.
  • Uso del Símbolo de Dólar ($): El símbolo de dólar es válido en los nombres de variables y se utiliza frecuentemente en bibliotecas como jQuery para identificar elementos del DOM. Ejemplo: elemento$, miVariable$.
  • Nombres descriptivos: Es importante usar nombres de variables que sean descriptivos y representen claramente el valor que almacenan. Ejemplo: nombreUsuario, cantidadTotal, edadCliente.
  • Evitar caracteres especiales: No utilices caracteres especiales como !, @, %, etc. Solo se permiten letras, números, guiones bajos (_) y símbolos de dólar ($).
  • Empieza con una letra: Los nombres de las variables deben empezar con una letra, un guion bajo (_) o el símbolo de dólar ($). No deben comenzar con un número.
  • Evitar palabras reservadas: No utilices palabras reservadas de JavaScript como let, const, var, if, function, class, etc., como nombres de variables.

Siguiendo estas buenas prácticas, puedes escribir código más limpio, legible y mantenible.

Operadores

Los operadores permiten realizar operaciones con valores y variables. Se clasifican en diferentes tipos:

Operadores aritméticos

Se usan para realizar cálculos matemáticos.

  • + (Suma): 5 + 3 // 8
  • - (Resta): 10 - 4 // 6
  • * (Multiplicación): 6 * 2 // 12
  • / (División): 8 / 2 // 4
  • % (Módulo o residuo): 10 % 3 // 1
  • ** (Exponente): 2 ** 3 // 8

Operadores de asignación

Se usan para asignar valores a las variables.

  • = (Asignación): x = 10
  • += (Suma y asignación): x += 5 // x = x + 5
  • -= (Resta y asignación): x -= 2 // x = x - 2
  • *= (Multiplicación y asignación): x *= 3 // x = x * 3
  • /= (División y asignación): x /= 4 // x = x / 4

Operadores de comparación

Se usan para comparar valores y devuelven un resultado booleano (true o false).

  • == (Igualdad): 5 == "5" // true (compara solo el valor)
  • === (Igualdad estricta): 5 === "5" // false (compara valor y tipo de dato)
  • != (Diferente): 10 != 5 // true
  • !== (Diferente estricto): 10 !== "10" // true
  • > (Mayor que): 8 > 3 // true
  • < (Menor que): 4 < 7 // true
  • >= (Mayor o igual que): 6 >= 6 // true
  • <= (Menor o igual que): 3 <= 5 // true

Operadores lógicos

Se usan para combinar expresiones booleanas.

  • && (AND): true && false // false (ambos deben ser true para devolver true)
  • || (OR): true || false // true (al menos uno debe ser true)
  • ! (NOT): !true // false (niega el valor booleano)

Los operadores son esenciales para trabajar con condiciones y cálculos en JavaScript.

Strings

Un string es una cadena de caracteres utilizada para representar texto. Se puede definir usando comillas simples (' '), comillas dobles (" ") o template literals (` `).

Concatenación de Strings

La concatenación permite unir dos o más strings. Se puede hacer de dos formas:

  • Usando el operador +: "Hola" + " mundo": "Hola mundo"
  • Usando Template Literals: Se utilizan comillas invertidas (`) y la sintaxis ${} para insertar variables.

Métodos útiles para Strings

JavaScript proporciona varios métodos para manipular strings:

  • .length: Devuelve la longitud del string.
  • .toUpperCase(): Convierte el texto a mayúsculas.
  • .toLowerCase(): Convierte el texto a minúsculas.
  • .trim(): Elimina espacios en blanco al inicio y al final.
  • .includes("texto"): Verifica si una cadena contiene otra.
  • .slice(inicio, fin): Extrae parte de un string.
  • .replace("viejo", "nuevo"): Reemplaza parte de un string
  • .split("separador"): Divide un string en un array.

Estos métodos facilitan la manipulación de texto en JavaScript.

Números

En JavaScript, los números se representan con el tipo de dato Number. No hay diferencia entre enteros y decimales, ambos se manejan bajo el mismo tipo.

Operaciones con Números

Se pueden realizar operaciones matemáticas usando operadores aritméticos como + (suma), - (resta), * (multiplicación), / (división), % (módulo) y ** (exponente).

Métodos útiles para Números

JavaScript proporciona varios métodos para trabajar con números:

  • isInteger(numero): Verifica si un número es un entero.
  • parseInt(string): Convierte un string en un número entero.
  • parseFloat(string): Convierte un string en un número decimal.
  • toFixed(decimales): Redondea un número a la cantidad de decimales especificada.
  • toPrecision(dígitos): Formatea un número con una cantidad específica de dígitos significativos.
  • Math.round(numero): Redondea al entero más cercano.
  • Math.floor(numero): Redondea hacia abajo.
  • Math.ceil(numero): Redondea hacia arriba.
  • Math.random(): Genera un número decimal aleatorio entre 0 y 1.
  • Math.abs(numero): Devuelve el valor absoluto de un número.
  • Math.max(num1, num2, ...): Devuelve el número más grande de una lista.
  • Math.min(num1, num2, ...): Devuelve el número más pequeño de una lista.
  • Math.pow(base, exponente): Calcula una potencia.
  • Math.sqrt(numero): Calcula la raíz cuadrada de un número.

Estos métodos permiten realizar cálculos y manipular valores numéricos de forma sencilla en JavaScript.

(Type Casting) Conversión de Tipos de Datos

En JavaScript, es común tener que convertir valores de un tipo de dato a otro. Para ello, JavaScript ofrece varias funciones y métodos.

Conversión de String a Número

Si tenemos un valor de tipo string y queremos convertirlo a un número, podemos usar los siguientes métodos:

  • Number(valor): Convierte el valor a un número, si es posible. Si no puede convertirlo, devuelve NaN.
  • parseInt(valor): Convierte un string en un número entero. Si el string no comienza con un número, devuelve NaN.
  • parseFloat(valor): Convierte un string en un número decimal. Al igual que parseInt(), devuelve NaN si no puede realizar la conversión.

Conversión de Número a String

Para convertir un número a un string, se pueden usar los siguientes métodos:

  • toString(): Convierte un número a su representación en string. Ejemplo: 123.toString(): "123"
  • String(valor): Convierte cualquier tipo de valor (número, objeto, etc.) a string.

Conversión a Booleano

En JavaScript, ciertos valores se consideran "falsy" (falsos) y otros "truthy" (verdaderos). Para convertir cualquier valor a un booleano, puedes usar la función Boolean(valor), que devuelve:

  • Falsy: undefined, null, 0, NaN, "", false
  • Truthy: Cualquier valor que no sea falsy.

Ventanas de Diálogo

En JavaScript, se pueden utilizar ventanas de diálogo para interactuar con el usuario. Estas ventanas son proporcionadas por el navegador y permiten mostrar mensajes, recibir respuestas o confirmar acciones. Las principales ventanas de diálogo son alert(), confirm(), y prompt().

1. alert()

La función alert() muestra una ventana emergente con un mensaje y un botón "Aceptar". Esta ventana no devuelve ningún valor y simplemente sirve para mostrar información al usuario. Cuando aparece, detiene la ejecución del código hasta que el usuario haga clic en "Aceptar".

2. confirm()

La función confirm() muestra una ventana con un mensaje y dos botones: "Aceptar" y "Cancelar". Devuelve un valor booleano: true si el usuario hace clic en "Aceptar", y false si hace clic en "Cancelar". Este método se usa comúnmente para confirmar una acción o decisión del usuario, como la eliminación de un elemento.

3. prompt()

La función prompt() permite al usuario ingresar un valor en un campo de texto. Muestra un mensaje, un campo de entrada y dos botones: "Aceptar" y "Cancelar". El valor ingresado por el usuario se devuelve como un string. Si el usuario hace clic en "Cancelar", se devuelve null.

4. Otros tipos de ventanas emergentes (no estándar)

Algunos navegadores también ofrecen otras funciones para mostrar ventanas emergentes, aunque no son tan comunes como las anteriores. Estas incluyen:

  • console.log(): Aunque no es una ventana emergente, se utiliza para mostrar información en la consola del navegador, útil para depuración.
  • window.open(): Abre una nueva ventana del navegador con una URL específica. Es comúnmente usada para abrir páginas en nuevas pestañas o ventanas.

Impacto en el flujo de ejecución

Las funciones alert(), confirm(), y prompt() interrumpen el flujo de ejecución de la página. Cuando una de estas ventanas aparece, detiene el código JavaScript hasta que el usuario haya interactuado con la ventana, lo que puede resultar en una mala experiencia de usuario si se usan en exceso. En general, es recomendable utilizarlas con moderación y considerar alternativas como modales o notificaciones personalizadas cuando sea necesario.

Selección de Elementos del DOM

En JavaScript, se puede seleccionar elementos del DOM (Document Object Model) para poder interactuar con ellos. Existen varias funciones dentro del objeto document que permiten seleccionar elementos de la página web para luego modificarlos o extraer información. A continuación se describen las funciones más comunes.

1. document.getElementById()

La función getElementById() permite seleccionar un elemento del DOM usando su identificador único (id). Esta función devuelve un solo elemento, ya que id debe ser único dentro de la página.

2. document.getElementsByClassName()

La función getElementsByClassName() selecciona todos los elementos que tienen una clase específica. Esta función devuelve una colección de nodos (una lista de elementos) que coinciden con la clase dada. Es importante notar que devuelve una colección en vivo, lo que significa que si se agregan o eliminan elementos con esa clase, la colección se actualizará automáticamente.

3. document.getElementsByTagName()

La función getElementsByTagName() selecciona todos los elementos del DOM que coinciden con una etiqueta HTML específica. Al igual que getElementsByClassName(), devuelve una colección de nodos en vivo.

4. document.querySelector()

La función querySelector(), selecciona el primer elemento del DOM que coincide con un selector CSS especificado. A diferencia de getElementById(), que selecciona por ID, esta función permite usar selectores más flexibles, como clases (.clase), IDs (#id) o etiquetas (etiqueta). Devuelve un único nodo, incluso si hay múltiples coincidencias, y la colección no es en vivo.

5. document.querySelectorAll()

querySelectorAll() selecciona todos los elementos que coinciden con un selector CSS especificado, devolviendo una NodeList estática (no en vivo). Esto significa que, a diferencia de getElementsByClassName() o getElementsByTagName(), si el DOM cambia después de la selección, la lista no se actualizará automáticamente. Se pueden usar métodos como .forEach() para recorrer los elementos de la lista.

Impacto en el flujo

Las funciones como getElementById(), getElementsByClassName(), y getElementsByTagName() permiten manipular elementos de forma directa. Al seleccionar un elemento, se puede modificar su contenido, estilo, atributos, entre otros. Es fundamental entender cómo funcionan las colecciones de nodos devueltas (en vivo o no en vivo) para evitar problemas de actualización y manipulación incorrecta de los elementos.

Modificación del contenido en el DOM

El DOM (Document Object Model) permite manipular dinámicamente el contenido de una página web. JavaScript ofrece múltiples métodos para modificar el texto, insertar nuevos elementos y eliminar nodos dentro del documento HTML.

1. Modificar el texto de un elemento

Existen diferentes formas de cambiar el contenido textual de un elemento:

  • textContent: Modifica o devuelve el contenido de texto sin interpretar etiquetas HTML.
  • innerHTML: Modifica el contenido del elemento e interpreta etiquetas HTML.
  • innerText: Similar a textContent, pero respeta los estilos CSS como display: none.

2. Insertar nuevos elementos en el DOM

JavaScript permite agregar nuevos elementos dentro del DOM utilizando diferentes métodos:

  • appendChild(): Agrega un nuevo elemento dentro de otro como su hijo.
  • prepend(): Inserta un nuevo elemento como el primer hijo de un contenedor.
  • insertAdjacentHTML(): Inserta contenido HTML en distintas posiciones respecto a un elemento ("beforebegin", "afterbegin", "beforeend", "afterend").

3. Eliminar contenido o elementos del DOM

Para eliminar elementos del DOM, se pueden utilizar los siguientes métodos:

  • removeChild(): Elimina un elemento hijo dentro de otro.
  • remove(): Borra directamente un elemento del DOM.
  • replaceChild(): Reemplaza un nodo hijo por otro.

Estos métodos permiten modificar dinámicamente el contenido de una página web, ya sea cambiando texto, insertando elementos o eliminando contenido de manera eficiente.

Estructuras de Control: `if`, `if else` y `else`

En JavaScript, la estructura de control if permite ejecutar un bloque de código solo si se cumple una condición específica. Si la condición no se cumple, se pueden usar else y else if para manejar otros casos posibles.

En JavaScript, la estructura de control if permite ejecutar un bloque de código solo si se cumple una condición específica. Si la condición no se cumple, se pueden usar else y else if para manejar otros casos posibles.

1. if: Evaluando una Condición

La sentencia if se usa para ejecutar código solo si una condición es verdadera.

Sintaxis:


if (condición) {
    // Código.
}

2. if else: Manejo de Condiciones Alternativas

Cuando se necesita ejecutar un bloque de código en caso de que la condición no se cumpla, se usa else.

Sintaxis:


if (condición) {
    // Código...
} else {
    // Código...
}

3. else if: Evaluando Múltiples Condiciones

Si se tienen varias condiciones posibles, se puede usar else if para evaluar diferentes escenarios antes de llegar al else.

Sintaxis:


if (condición1) {
    // Código...
} else if (condición2) {
    // Código...
} else {
    // Código...
}

Consideraciones Importantes

  • Las condiciones dentro de if deben evaluarse a un valor true o false.
  • El bloque else es opcional, pero se recomienda para manejar casos no contemplados.
  • Se pueden anidar múltiples else if, pero es importante no abusar de ellos para evitar código difícil de leer.
  • Es recomendable utilizar operadores lógicos como && (AND), || (OR) y ! (NOT) para combinar condiciones dentro de un mismo if.

Las estructuras if, else if y else son fundamentales en la toma de decisiones dentro de un programa en JavaScript.

Estructura de Control: switch

En JavaScript, la sentencia switch se usa para tomar decisiones basadas en el valor de una expresión. En lugar de evaluar múltiples condiciones con if else, el switch compara un valor con diferentes casos y ejecuta el código correspondiente.

1. Sintaxis de switch

Ejemplo de uso:


switch (expresión) {
    case valor1:
        // Código a ejecutar si expresión === valor1
    break;
    case valor2:
        // Código a ejecutar si expresión === valor2
    break;
    default:
        // Código a ejecutar si ningún caso coincide
}

2. Funcionamiento

  • La expresión dentro de switch se evalúa y se compara con los valores de cada case.
  • Si hay una coincidencia, se ejecuta el código del caso correspondiente.
  • La sentencia break evita que la ejecución continúe en los siguientes casos.
  • El bloque default es opcional y se ejecuta si no hay coincidencias.

3. Ejemplo Práctico


let dia = "lunes";

switch (dia) {
    case "lunes":
        console.log("Inicio de semana.");
    break;
    case "viernes":
        console.log("Fin de semana cerca.");
    break;
    case "sábado":
    case "domingo":
        console.log("Es fin de semana.");
    break;
    default:
        console.log("Día no reconocido.");
}

4. Consideraciones Importantes

  • El uso de break es importante para evitar la ejecución de múltiples casos.
  • Si dos casos deben ejecutar el mismo código, pueden agruparse sin break entre ellos.
  • El switch solo compara valores con === (estrictamente iguales).
  • Para condiciones más complejas, if else puede ser una mejor opción.

El uso de switch ayuda a organizar el código y mejorar la legibilidad cuando hay múltiples comparaciones posibles.

Bucle for

El bucle for es una estructura de control en JavaScript que permite ejecutar un bloque de código un número determinado de veces. Se compone de tres partes: inicialización, condición y actualización.

1. Sintaxis del bucle for

El bucle for sigue la siguiente estructura:


for (inicialización; condición; actualización) {
    // Código a ejecutar en cada iteración
}

2. Explicación de los parámetros

  • Inicialización: Se ejecuta una sola vez antes de que el bucle comience. Generalmente, se usa para declarar e inicializar una variable de control.
  • Condición: Se evalúa antes de cada iteración. Si es true, el bloque de código se ejecuta; si es false, el bucle termina.
  • Actualización: Se ejecuta al final de cada iteración y generalmente se usa para modificar la variable de control.

3. Ejemplo de uso

Este ejemplo muestra un bucle for que imprime los números del 1 al 5 en la consola: 


for (let i = 1; i <= 5; i++) {
    console.log(i);
}

4. Consideraciones

  • Si la condición nunca se cumple, el bucle no se ejecuta.
  • Si no hay una actualización correcta de la variable de control, el bucle puede volverse infinito.
  • Se puede usar la palabra clave break para salir del bucle antes de que la condición sea falsa.
  • La palabra clave continue permite saltar una iteración sin salir del bucle.

5. Uso de break y continue

Ejemplo con break: el bucle se detiene cuando i vale 3.


for (let i = 1; i <= 5; i++) {
    if (i === 3) {
        break; // Sale del bucle cuando i es 3
    }
    console.log(i);
}

Ejemplo con continue: la iteración se salta cuando i es 3.


for (let i = 1; i <= 5; i++) {
    if (i === 3) {
        continue; // Salta la iteración cuando i es 3
    }
    console.log(i);
}

El bucle for es una herramienta esencial en JavaScript para recorrer estructuras como arrays, listas y ejecutar tareas repetitivas de manera controlada.

Bucle while

El bucle while es una estructura de control que ejecuta un bloque de código mientras una condición especificada sea true. A diferencia del bucle for, no requiere una inicialización ni una actualización dentro de su estructura.

1. Sintaxis del bucle while

La estructura básica del bucle while es la siguiente:


while (condición) {
    // Código a ejecutar mientras la condición sea verdadera
}

2. Funcionamiento

  • Primero, se evalúa la condición.
  • Si la condición es true, el bloque de código se ejecuta.
  • Después de ejecutar el código, la condición se vuelve a evaluar.
  • El proceso se repite hasta que la condición se vuelva false.

3. Ejemplo de uso

Este ejemplo usa un bucle while para imprimir los números del 1 al 5 en la consola: 


let i = 1;
while (i <= 5) {
    console.log(i);
    i++; // Se incrementa la variable de control
}

4. Peligro de bucles infinitos

Si la condición nunca se vuelve false, el bucle while se ejecutará indefinidamente.


// ❌ Esto causará un bucle infinito, ya que la condición siempre es verdadera
while (true) {
    console.log("Esto nunca terminará");
}

5. Uso de break y continue

Se puede usar break para salir del bucle antes de que la condición sea falsa. 


let i = 1;
while (i <= 5) {
    if (i === 3) {
        break; // Sale del bucle cuando i es 3
    }
    console.log(i);
    i++;
}

También se puede usar continue para saltar una iteración.


let i = 0;
while (i < 5) {
    i++;
    if (i === 3) {
        continue; // Salta la iteración cuando i es 3
    }
    console.log(i);
}

El bucle while es útil cuando no se conoce de antemano cuántas veces se ejecutará el código, como en validaciones o lecturas de datos dinámicos.

Funciones

En JavaScript, las funciones son bloques de código reutilizables que pueden ejecutarse cuando se necesiten.

1. Definición de una Función

Para definir una función en JavaScript, se usa la palabra clave function, seguida del nombre de la función, paréntesis de parámetros () y un bloque de código dentro de llaves {}.

Sintaxis:


function nombreDeLaFuncion() {
    // Código que se ejecutará cuando se llame a la función
}

2. Llamado de una Función

Para ejecutar una función, simplemente se escribe su nombre seguido de paréntesis ().

Ejemplo:


function saludar() {
    console.log("¡Hola, bienvenido a JavaScript!");
}

saludar(); // Llamamos a la función

3. Funciones con Parámetros

Podemos pasar valores a una función usando parámetros. Estos valores se reciben dentro de los paréntesis y pueden usarse dentro de la función.

Ejemplo:


function sumar(a, b) {
    console.log("La suma es:", a + b);
}

sumar(5, 3); // Resultado: La suma es: 8

4. Funciones con Valor de Retorno

Las funciones pueden devolver un valor usando la palabra clave return. Esto permite que el resultado se almacene o se use en otro lugar del programa.

Ejemplo:


function multiplicar(x, y) {
    return x * y; // Devuelve el resultado de la multiplicación
}

let resultado = multiplicar(4, 6);
console.log("El resultado es:", resultado); // El resultado es: 24

5. Funciones como Expresiones

También es posible definir funciones dentro de variables. A esto se le llama expresión de función.

Ejemplo:


let dividir = function(a, b) {
    return a / b;
};

console.log(dividir(10, 2)); // Resultado: 5

Las funciones son fundamentales en JavaScript y permiten estructurar mejor el código, evitando la repetición y mejorando la reutilización.

Funciones Flecha (Arrow Functions)

Las funciones flecha (Arrow Functions) en JavaScript son una forma más compacta de escribir funciones. Se introdujeron en ES6 y ofrecen una sintaxis más limpia y moderna.

1. Sintaxis de una Arrow Function

En lugar de usar la palabra clave function, las funciones flecha se definen con el operador =>.

Ejemplo:


// Función clásica
function saludar() {
    return "¡Hola, mundo!";
}

// Función flecha equivalente
const saludar = () => "¡Hola, mundo!";
console.log(saludar()); // ¡Hola, mundo!

2. Arrow Functions con Parámetros

Si la función recibe parámetros, se colocan dentro de los paréntesis ().

Ejemplo:


// Función clásica
function sumar(a, b) {
    return a + b;
}

// Función flecha equivalente
const sumar = (a, b) => a + b;
console.log(sumar(5, 3)); // Resultado: 8

3. Arrow Functions con un Solo Parámetro

Si hay un solo parámetro, se pueden omitir los paréntesis.

Ejemplo:


    const cuadrado = num => num * num;
    console.log(cuadrado(4)); // Resultado: 16

4. Arrow Functions con Múltiples Líneas

Si la función tiene más de una línea de código, se deben usar llaves {} y la palabra clave return.

Ejemplo:


const multiplicar = (x, y) => {
    let resultado = x * y;
    return resultado;
};

console.log(multiplicar(4, 6)); // Resultado: 24

5. Diferencias entre Arrow Functions y Funciones Clásicas

Aunque las funciones flecha pueden reemplazar a las clásicas en muchos casos, tienen diferencias importantes:

  • No tienen su propio this: En las funciones flecha, this se mantiene del contexto en el que se creó la función. En cambio, en las funciones clásicas, this cambia según cómo se llama la función.
  • No tienen arguments: No se puede acceder al objeto arguments dentro de una función flecha, lo que puede ser un problema si necesitas trabajar con argumentos dinámicos.
  • No pueden usarse como métodos de objetos: Si usas una arrow function dentro de un objeto, this no apuntará al objeto, sino al contexto exterior.
  • No pueden usarse como constructores: No se pueden usar con new para crear instancias de objetos.

6. Ejemplo de this en Arrow Functions vs. Funciones Clásicas

Ejemplo con función clásica:


const objeto = {
    nombre: "Ejemplo",
    mostrar: function() {
        console.log(this.nombre); // "Ejemplo"
    }
};

objeto.mostrar();

Ejemplo con Arrow Function (Error):


const objeto2 = {
    nombre: "Ejemplo",
    mostrar: () => {
        console.log(this.nombre); // undefined, porque this apunta al contexto global
    }
};

objeto2.mostrar();

Las funciones flecha son ideales para callbacks, métodos en arrays como map() y filter(), y situaciones donde no necesitas un this específico.

Arrays

Un array en JavaScript es una estructura de datos que permite almacenar múltiples valores en una sola variable. Los arrays pueden contener cualquier tipo de dato, incluidos números, strings, booleanos, objetos e incluso otros arrays.

1. Crear un Array

Se pueden crear arrays de diferentes formas:


// Usando corchetes []
let frutas = ["Manzana", "Banana", "Cereza"];

// Usando el constructor Array()
let numeros = new Array(1, 2, 3, 4, 5);

2. Acceder a Elementos de un Array

Para acceder a los elementos de un array, se usa un índice (que empieza en 0).


let colores = ["Rojo", "Verde", "Azul"];
console.log(colores[0]); // "Rojo"
console.log(colores[2]); // "Azul"

3. Modificar Elementos

Los elementos de un array pueden modificarse accediendo a su índice.


let numeros = [10, 20, 30];
numeros[1] = 25; 
console.log(numeros); // [10, 25, 30]

4. Métodos Comunes de los Arrays

JavaScript proporciona varios métodos para manipular arrays:

  • push(): Agrega un elemento al final.
  • pop(): Elimina el último elemento.
  • shift(): Elimina el primer elemento.
  • unshift(): Agrega un elemento al inicio.
  • splice(): Agrega o elimina elementos en una posición específica.
  • slice(): Extrae una porción del array sin modificarlo.
  • indexOf(): Busca la posición de un elemento.
  • includes(): Verifica si un elemento está en el array.

5. Recorrer un Array

Se pueden recorrer los arrays usando bucles:


let numeros = [1, 2, 3, 4, 5];

// Usando for clásico
for (let i = 0; i < numeros.length; i++) {
    console.log(numeros[i]);
}

// Usando forEach()
numeros.forEach(num => console.log(num));

6. Arrays Multidimensionales

Un array puede contener otros arrays, creando una estructura de múltiples dimensiones. 


let matriz = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
];

console.log(matriz[1][2]); // 6

7. Transformación y filtrado

  • map(): Crea un nuevo array aplicando una función a cada elemento.
  • filter(): Crea un nuevo array con los elementos que cumplen una condición.
  • forEach(): Ejecuta una función para cada elemento del array. 

// Ejemplo de uso
let numeros = [1, 2, 3, 4, 5];

// Multiplicar cada número por 2
let duplicados = numeros.map(num => num * 2); // [2, 4, 6, 8, 10]

// Filtrar números mayores a 2
let mayoresDos = numeros.filter(num => num > 2); // [3, 4, 5]

// Imprimir cada número en consola
numeros.forEach(num => console.log(num));

Los arrays son una estructura fundamental en JavaScript y se usan en una gran variedad de aplicaciones.

Objetos

teoria: MDN

Un objeto en JavaScript es una estructura de datos que almacena información en pares clave-valor.
A diferencia de los arrays, los objetos no tienen un orden específico y permiten organizar los datos de manera estructurada.

Creación de un Objeto

Existen varias maneras de crear un objeto:


// Objeto literal (forma más común)
let persona = {
    nombre: "Juan",
    edad: 30,
    profesion: "Desarrollador"
};

// Usando new Object() (menos común)
let auto = new Object();
auto.marca = "Toyota";
auto.modelo = "Corolla";
auto.año = 2022;

// Creación dinámica de propiedades
let usuario = {};
usuario.nombre = "Carlos";
usuario.edad = 25;

Acceder a Propiedades

Podemos acceder a las propiedades de dos maneras:


console.log(persona.nombre);  // "Juan" (notación de punto)
console.log(persona["edad"]); // 30 (notación de corchetes)

Modificar y Eliminar Propiedades


persona.edad = 31;  // Modificar una propiedad
delete persona.profesion;  // Eliminar una propiedad

Métodos en Objetos

Un objeto puede contener funciones, llamadas métodos, para realizar acciones con sus propiedades.


let usuario = {
    nombre: "Ana",
    saludo() {
        return "Hola, mi nombre es " + this.nombre;
    }
};
console.log(usuario.saludo());

Métodos útiles para Objetos

  • Object.keys(objeto): Devuelve un array con las claves del objeto.
  • Object.values(objeto): Devuelve un array con los valores.
  • Object.entries(objeto): Devuelve un array de pares clave-valor.
  • hasOwnProperty(propiedad): Verifica si una propiedad existe en el objeto.

Ejemplo con Métodos


let producto = {
    nombre: "Laptop",
    precio: 1200,
    disponible: true
};
                    
console.log(Object.keys(producto));
// ["nombre", "precio", "disponible"]

console.log(Object.values(producto));
// ["Laptop", 1200, true]

console.log(Object.entries(producto));
// [["nombre", "Laptop"], ["precio", 1200], ["disponible", true]]

Copiar y Fusionar Objetos

Para copiar o combinar objetos, podemos usar el operador spread (...).


// Copia de un objeto
let copiaProducto = { ...producto };
console.log(copiaProducto);

// Fusionar objetos
let detallesExtra = { marca: "Dell", peso: "1.5kg" };
let productoCompleto = { ...producto, ...detallesExtra };
console.log(productoCompleto);

Convertir Objetos a JSON

JavaScript permite convertir objetos a JSON para enviarlos o almacenarlos.


let jsonProducto = JSON.stringify(producto);
console.log(jsonProducto);

// Convertir JSON a objeto
let objetoProducto = JSON.parse(jsonProducto);
console.log(objetoProducto);

JSON

JSON (JavaScript Object Notation) es un formato ligero de intercambio de datos. Es fácil de leer y escribir para los humanos y sencillo de procesar para las máquinas, lo que lo convierte en un estándar en comunicación entre aplicaciones.

Casos de uso

  • Intercambiar datos entre un cliente (navegador) y un servidor.
  • Almacenar configuraciones en archivos de manera estructurada.
  • Representar estructuras de datos en APIs y bases de datos.
  • Serializar y deserializar información en aplicaciones web y móviles.

Estructura de JSON

JSON está compuesto por pares clave-valor, similar a un objeto en JavaScript, pero con claves siempre en formato de cadena:

{
    "nombre": "Carlos",
    "edad": 28,
    "ciudad": "Madrid",
    "hobbies": ["leer", "viajar", "fútbol"],
    "activo": true
}

Reglas importantes de JSON:

  • Las claves deben estar entre comillas dobles ("").
  • Los valores pueden ser: cadenas, números, booleanos, arreglos, objetos o null.
  • No se permiten funciones o variables en JSON, solo datos estáticos.
Métodos para trabajar con JSON
  • JSON.stringify(objeto): Convierte un objeto de JavaScript en una cadena JSON.
  • JSON.parse(textoJSON): Convierte una cadena JSON en un objeto JavaScript.

Ejemplo práctico

// Objeto en JavaScript
let usuario = {
    nombre: "Laura",
    edad: 25,
    ciudad: "Barcelona",
    activo: true
};

// Convertir a JSON
let usuarioJSON = JSON.stringify(usuario);
console.log(usuarioJSON); 
// Salida: '{"nombre":"Laura","edad":25,"ciudad":"Barcelona","activo":true}'

// Convertir JSON a objeto
let nuevoObjeto = JSON.parse(usuarioJSON);
console.log(nuevoObjeto.nombre); // "Laura"
console.log(nuevoObjeto.activo); // true

Convertir un array de objetos a JSON

let productos = [
    { nombre: "Laptop", precio: 1200, stock: true },
    { nombre: "Mouse", precio: 25, stock: false }
];

// Convertir array de objetos a JSON
let productosJSON = JSON.stringify(productos);
console.log(productosJSON);

// Convertir JSON de vuelta a array de objetos
let productosParseados = JSON.parse(productosJSON);
console.log(productosParseados[0].nombre); // "Laptop"

Uso de JSON con Fetch API

JSON se usa comúnmente en peticiones HTTP para enviar y recibir datos.

// Obtener datos de una API y convertirlos a un objeto JavaScript
fetch("https://jsonplaceholder.typicode.com/users")
    .then(response => response.json()) // Convertir JSON a objeto
    .then(data => console.log(data))   // Trabajar con los datos
    .catch(error => console.error("Error:", error));

Estados de carga de una página

Cuando una página web se carga en el navegador, pasa por diferentes estados. Conocer estos estados es clave para saber en qué momento se puede manipular el DOM de manera segura y eficiente.

1. Estados de carga de un documento

El navegador sigue un flujo específico al cargar una página web, y podemos identificar estos estados con document.readyState:

  • loading: El navegador está cargando el HTML inicial. Aún no ha construido completamente el DOM.
  • interactive: El DOM ha sido construido, pero los recursos externos (CSS, imágenes, scripts) pueden seguir cargando.
  • complete: Toda la página, incluidos los recursos externos, han terminado de cargarse.
console.log(document.readyState);

2. Eventos clave durante la carga de la página

DOMContentLoaded

Se dispara cuando el HTML ha sido completamente cargado y el DOM ha sido construido, pero sin esperar a imágenes, CSS u otros recursos externos.

document.addEventListener("DOMContentLoaded", () => {
    console.log("El DOM ha sido completamente cargado");
});

load

Se dispara cuando la página ha terminado de cargar completamente, incluyendo imágenes, hojas de estilo y otros recursos.

window.addEventListener("load", () => {
    console.log("Todos los recursos han terminado de cargarse.");
});

beforeunload

Se dispara antes de que el usuario salga de la página. Se usa para mostrar advertencias antes de cerrar o recargar.

window.addEventListener("beforeunload", (event) => {
    event.preventDefault();
    event.returnValue = "¿Seguro que quieres salir?";
});

3. Dónde comenzar a manipular el DOM

  • Si solo necesitas acceder a elementos HTML y modificar el DOM: usa DOMContentLoaded.
  • Si necesitas esperar a que las imágenes y estilos estén listos: usa load.
  • Si necesitas realizar algo antes de que el usuario salga de la página: usa beforeunload o unload.

4. Ejemplo completo

document.addEventListener("DOMContentLoaded", () => {
    console.log("El DOM está listo");
    document.querySelector("h1").style.color = "blue";
});

window.addEventListener("load", () => {
    console.log("Todos los recursos han cargado");
    document.querySelector("img").style.border = "2px solid red";
});

window.addEventListener("beforeunload", (event) => {
    event.preventDefault();
    event.returnValue = "¿Seguro que quieres salir?";
});

Conclusión

  • DOMContentLoaded: Mejor momento para empezar a manipular el DOM.
  • load: Útil cuando necesitas que imágenes y CSS estén completamente cargados.
  • beforeunload: Útil para manejar eventos de salida de la página.
